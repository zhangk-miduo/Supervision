## 1. 技术选型原则

| 原则 | 说明 |
| --- | --- |
| 复用现有 IMP 技术栈 | 不引入新的技术体系，降低学习与维护成本 |
| 团队经验优先 | 选用团队已熟练的框架与组件 |
| 企业规范一致 | 符合现有工程规范（分层、命名、部署） |

## 6. 自动化执行引擎

- **定位**：轻量工作流引擎，承担「按序执行节点链」的职责。
- **不采用**：BPM、n8n、Activiti 等重型引擎（复杂度 / 维护成本高）。
- **执行模型**：Task → 有序 Node 列表 → 依次执行，前序节点结果可作为后续节点上下文。
- 临时上下文存放于 Redis：`task:context:{executionId}`。

---

## 7. Node 执行器（策略模式）

定义统一执行接口，各节点类型即插即用：

```java
public interface NodeExecutor {
    NodeResult execute(TaskNode node);
}
```

内置实现：

| 实现类 | 节点类型 | 职责 |
| --- | --- | --- |
| `HttpNodeExecutor` | HTTP | 调用外部接口查询数据 |
| `ConditionNodeExecutor` | Condition | 规则判断，决定分支 |
| `WechatNodeExecutor` | Wechat | 调用企微机器人发送通知 |

新增节点（SQL / Email / DingTalk / Feishu / AI）只需新增 `NodeExecutor` 实现并注册。

## 8. 定时任务（Quartz）

- **选型理由**：原生支持 Cron、Job 持久化、集群模式；相比 XXL-JOB 更轻、与 Spring 集成更自然。
- **调度表** `task_schedule`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| task_id | BIGINT | 关联任务 |
| cron_expression | VARCHAR | Cron 表达式 |
| status | TINYINT | 启用 / 停用 |

- Quartz 触发时加载任务 → 获取节点链 → 调用执行引擎。

## 9. 节点技术设计

### 9.1 HTTP 节点
`config_json` 示例：
```json
{
  "type": "http",
  "url": "https://api.example.com/status",
  "method": "GET",
  "headers": { "Authorization": "Bearer xxx" }
}
```
执行结果统一封装为 `NodeResult`（status / data / message）。

### 9.2 Condition 节点
配置：`field` / `operator` / `value`。
支持运算符：`=`, `!=`, `>`, `<`, `>=`, `<=`, `contains`, `empty`。
用于根据 HTTP 节点返回数据决定是否继续 / 通知。

### 9.3 企业微信节点
配置：`robotId` / `template`。
发送链路：`WechatNodeExecutor` → `WechatClient` → Webhook → 企业微信群。
支持发送前「测试」验证连通性。

## 10. 数据库设计（MySQL 8）

> 以下 DDL 基于对话中描述的字段重建，供实现参考。

```sql
-- 督办任务
CREATE TABLE supervision_task (
    id          BIGINT       PRIMARY KEY AUTO_INCREMENT,
    name        VARCHAR(128) NOT NULL COMMENT '任务名称',
    description VARCHAR(512) DEFAULT NULL COMMENT '任务描述',
    status      TINYINT      NOT NULL DEFAULT 1 COMMENT '1启用 0停用',
    schedule_type TINYINT    NOT NULL DEFAULT 0 COMMENT '0手动 1定时',
    created_by  VARCHAR(64)  DEFAULT NULL,
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) COMMENT='督办任务';

-- 流程节点
CREATE TABLE supervision_task_node (
    id        BIGINT       PRIMARY KEY AUTO_INCREMENT,
    task_id   BIGINT       NOT NULL COMMENT '关联任务',
    node_type VARCHAR(32)  NOT NULL COMMENT 'http/condition/wechat',
    node_order INT         NOT NULL COMMENT '执行顺序',
    config    JSON         NOT NULL COMMENT '节点配置',
    created_at DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP
) COMMENT='流程节点';

-- 执行记录
CREATE TABLE supervision_task_execution (
    id         BIGINT       PRIMARY KEY AUTO_INCREMENT,
    task_id    BIGINT       NOT NULL COMMENT '关联任务',
    status     TINYINT      NOT NULL COMMENT '0成功 1失败 2执行中',
    result     TEXT         DEFAULT NULL COMMENT '执行结果',
    start_time DATETIME     NOT NULL,
    end_time   DATETIME     DEFAULT NULL
) COMMENT='执行记录';

-- 企业微信机器人
CREATE TABLE supervision_wechat_robot (
    id          BIGINT       PRIMARY KEY AUTO_INCREMENT,
    robot_id    VARCHAR(64)  NOT NULL COMMENT '机器人标识',
    name        VARCHAR(128) NOT NULL COMMENT '机器人名称',
    webhook_url VARCHAR(512) NOT NULL COMMENT 'Webhook 地址',
    template    VARCHAR(512) DEFAULT NULL COMMENT '消息模板',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
) COMMENT='企业微信机器人';

-- 调度配置
CREATE TABLE supervision_task_schedule (
    id              BIGINT       PRIMARY KEY AUTO_INCREMENT,
    task_id         BIGINT       NOT NULL COMMENT '关联任务',
    cron_expression VARCHAR(64)  NOT NULL COMMENT 'Cron 表达式',
    status          TINYINT      NOT NULL DEFAULT 1 COMMENT '1启用 0停用'
) COMMENT='调度配置';
```

## 11. Redis 设计

| Key | 用途 | 说明 |
| --- | --- | --- |
| `supervision:task:lock:{taskId}` | 分布式锁 | 防止同一任务并发执行 |
| `supervision:task:context:{executionId}` | 临时上下文 | 存放节点间传递的执行上下文，设过期时间 |

---

## 12. RabbitMQ 设计

- **目的**：异步发送企业微信通知，解耦执行与通知，提升主流程响应速度。
- **Queue**：`supervision.notification.queue`
- **Exchange**：`supervision.exchange`
- 执行引擎将通知消息投递队列 → `WechatSender` 消费并调用 Webhook。
