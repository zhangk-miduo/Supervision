## 2026-07-22 任务目标群不可选调查

- 用户反馈：机器人管理中已添加机器人，但创建任务时目标群列表为空。
- 静态链路确认：任务页面同时请求人员列表和机器人列表，并在 `Promise.all` 全部成功后才分别写入页面状态；目标群下拉使用机器人接口返回的 `records`。
- 服务器证据：机器人旧表存在启用记录，且加密 Webhook 配置表存在相同 ID 的启用记录；机器人数据并非缺失。
- 根因证据：`/persons` 查询返回 500，日志显示生产库 `supervision_person` 缺少当前模型要求的 `tenant_key` 字段。V3 迁移只执行 `CREATE TABLE IF NOT EXISTS`，无法升级已存在的旧表。人员请求失败使整个 `Promise.all` 被拒绝，机器人结果也未写入，最终表现为目标群不可选。
- 风险：旧部门表可能存在同类结构不兼容；人员和机器人两个独立数据源被前端强耦合，任一接口故障都会遮蔽另一个功能；页面首次挂载后新增机器人也不会自动刷新。
- 建议后续动作：新增兼容迁移以对齐旧人员/部门表结构；任务页面改为独立加载并分别显示错误；在打开创建向导或进入消息触达步骤时刷新目标群；仅允许选择启用且具有有效加密 Webhook 配置的目标。
- 验收重点：人员接口失败时目标群仍可选择；新添加机器人无需整页刷新即可出现；禁用或缺失加密配置的机器人不可选择；保存后的 `webhookId` 与实际发送配置一致。
- 本轮为 OpenSpec 探索，仅完成诊断与方案收敛，未修改业务代码。
