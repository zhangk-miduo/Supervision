# 实施任务：账号数据隔离与公开机器人共享

## 1. 数据库迁移与历史归属

- [x] 1.1 新增下一版本 Flyway 迁移，为任务、执行、逻辑群和 webhook 增加 `owner_account_id`，为 webhook 增加默认关闭的 `is_public`，并建立数据范围索引。
- [x] 1.2 调整逻辑群唯一约束为 `(tenant_key, owner_account_id, group_name)`，验证 webhook 与逻辑群所有者一致。
- [x] 1.3 按旧任务 `created_by = account.username` 精确回填任务归属，并从任务回填执行归属。
- [x] 1.4 制定并执行旧机器人、无法匹配任务和孤立执行的归属报告；不确定数据保持未归属且仅管理员可见。
- [ ] 1.5 增加旧库和当前库迁移测试，验证记录数、索引、默认私有、重复群名及不删除历史数据。

## 2. 统一数据范围基础设施

- [x] 2.1 从认证会话构造不可伪造的 `DataScope(accountId, isAdmin)`，并在控制器到应用服务之间显式传递。
- [x] 2.2 实现统一的读取范围、所有者断言和机器人可用性断言，定义资源不可见、共享收回和停用的稳定错误码。
- [x] 2.3 将任务、机器人和执行实体直出替换为安全 DTO，关联返回创建账号、显示名称、`ownedByCurrentUser`、`canEdit` 等字段。
- [x] 2.4 增加管理员专用创建人筛选并保证普通账号不能借筛选参数扩大范围。
- [x] 2.5 增加跨账号直接 ID、分页总数、404 防枚举和管理员只读全量的自动化测试。

## 3. 机器人归属与公开共享

- [x] 3.1 创建机器人时从会话写入 webhook 和逻辑群所有者，默认 `is_public=false`，禁止客户端指定其他所有者。
- [x] 3.2 重构机器人管理列表为本人、公开可用和管理员全部视图，并为公开非本人记录返回最小非敏感摘要。
- [x] 3.3 重构可选机器人查询，限定为启用、可解密且“本人所有或公开”的 webhook。
- [x] 3.4 为编辑、公开切换、停用、删除和机器人配置测试增加创建者所有权校验；管理员普通接口不得修改他人机器人。
- [x] 3.5 增加共享影响统计，在公开转私有或停用前返回其他账号启用任务引用数量。
- [x] 3.6 在任务保存、任务级测试发送和实际投递前重复校验机器人使用权；共享收回后停止发送并记录稳定脱敏失败原因。
- [x] 3.7 保持 per-webhook 限流，确保执行和 Delivery 可追踪实际调用任务账号。
- [x] 3.8 增加默认私有、公开选择、非所有者调用、非所有者修改拒绝、共享收回、停用和敏感字段保护测试。

## 4. 任务与执行日志隔离

- [x] 4.1 新建任务从会话写入 owner，移除 `TaskCreateRequest.createdBy` 的信任和持久化路径。
- [x] 4.2 为任务列表、详情、更新、删除和立即执行增加读取范围或所有者校验；管理员可读但不能通过普通接口变更他人任务。
- [x] 4.3 在手动执行、Quartz 执行和跳过记录创建时从任务复制 `owner_account_id`，手动触发另记 `triggered_by_account_id`。
- [x] 4.4 为执行列表、详情和投递明细增加独立执行归属过滤，不依赖任务仍然存在。
- [x] 4.5 验证任务删除后历史执行隔离、定时执行无 HTTP 会话、跳过执行和跨账号 execution ID 防越权。
