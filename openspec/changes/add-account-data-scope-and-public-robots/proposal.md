# 变更提案：账号数据隔离与公开机器人共享

## 背景

Supervision 已有登录账号和管理员角色，但机器人、任务、执行日志仍使用全局查询。任务的 `created_by` 是客户端可提交的字符串，机器人和执行日志没有可信账号归属，列表、详情、修改、删除、测试和立即执行也未按当前会话约束。任何已登录账号目前都可能看到或通过资源 ID 操作其他账号的数据。

业务需要让普通账号只管理自己的机器人、任务和执行日志，同时保留组织人员作为企业共享数据。管理员需要查看所有账号的数据并看到创建账号与名称，但全量查看不自动授予修改他人数据的权限。机器人还需要支持创建者主动公开：公开后其他账号可以在任务中调用，但不能修改、停用或读取敏感配置。

## 目标

- 为机器人、任务和执行日志建立由后端认证会话确定的稳定账号归属。
- 普通账号只能查看和操作自己的账号域数据，管理员可以查看所有账号域数据。
- 列表和详情关联展示创建账号、显示名称及必要的归属标识。
- 组织部门和人员保持企业共享，所有正常登录账号均可查询和选择。
- 机器人默认私有；创建者可公开具体消息推送，其他账号可用于任务发送但不能修改配置。
- 在列表、详情、修改、删除、测试、任务目标选择、立即执行和执行详情等所有入口统一执行服务端数据范围校验。
- 对历史数据提供确定、可审计且不造成跨账号泄露的归属迁移策略。

## 非目标

- 不把组织人员按登录账号隔离，也不以 `person_id` 作为业务数据归属键。
- 不实现部门级、角色级或指定账号白名单共享；首期仅支持私有/公开两种机器人可见性。
- 不赋予管理员普通编辑接口修改他人机器人或任务的能力；应急治理权限需要单独提出并审计。
- 不在本变更中重做角色操作权限矩阵；`ADMIN`、`OPERATOR`、`VIEWER` 的动作权限可在后续能力中细化。
- 不允许转移新建数据的创建者；历史数据认领只用于迁移补偿。
- 不暴露 Webhook URL、密文、key、Secret 或内部技术配置。

## 能力范围

1. `account-data-scope`：可信账号归属、普通账号隔离、管理员全量读取、创建人展示、执行归属继承、组织共享和历史数据迁移。
2. `wecom-messaging`：机器人私有/公开共享、创建者独占修改权、任务选择和执行期校验、共享收回及敏感字段保护。

## 关键决策

- 使用登录账号主键作为归属事实，字段统一为 `owner_account_id`；客户端提交的 `createdBy` 不再参与授权或归属写入。
- 管理员拥有全量读取范围，但任务、机器人及其配置的修改权仍归创建者；查看权限与变更权限分离。
- 执行记录在创建时复制任务的 `owner_account_id`，保证定时执行和任务删除后的历史日志仍可隔离。
- 首期在具体 webhook/消息推送上使用 `is_public`，默认 `false`。公开不扩散到同名逻辑群的其他推送。
- 其他账号只能取得公开机器人的非敏感选择器信息并用于任务发送；机器人管理配置测试仍仅限创建者。
- 任务保存、任务级测试发送和实际执行都重新校验机器人为“本人所有或公开可用”，不能只依赖前端隐藏按钮。

## 影响范围

- 数据库：扩展任务、执行、企微群和 webhook 的归属字段、公开标识、索引与历史回填。
- 后端：增加统一数据范围上下文和所有权断言；调整机器人、任务、执行、消息测试及调度执行链路。
- API：实体直出收敛为安全 DTO，增加创建账号展示字段、公开状态、可编辑标识及管理员创建人筛选。
- 前端：机器人分为“我创建的/公开可用”，三类数据展示创建账号与名称，并按后端能力隐藏操作。
- 测试：覆盖列表和直接 ID 越权、管理员只读全量、公开共享、共享收回、定时执行及敏感字段保护。
- 运维：新增 Flyway 迁移和历史数据归属检查，发布前必须备份数据库。

## 验收摘要

- 普通账号无法列出、读取或操作其他账号的私有机器人、任务和执行日志。
- 管理员能查看全部数据及创建账号/名称，但普通变更接口不能修改非自己创建的数据。
- 所有账号都能查询和选择共享组织人员。
- 公开机器人能被其他账号选入任务并发送；私有机器人不能被其他账号调用。
- 非创建者无法编辑、停用、执行机器人配置测试或取得公开机器人的敏感配置。
- 公开机器人收回或停用后，其他账号的既有任务不再发送，并形成可读、可审计的失败记录。
- 定时产生的执行日志归属于任务创建账号；任务删除后历史日志仍保持正确隔离。

## 风险与约束

- 机器人目前横跨旧机器人、webhook 和逻辑群模型，归属不一致会造成同名群跨账号污染，迁移必须统一处理。
- 公开机器人会让多个账号共享企微 webhook 的频率额度；现有限流必须继续以 webhook 为维度，并记录调用任务账号。
- 收回公开权限可能中断其他账号的定时任务，界面必须展示影响数量并要求确认。
- 旧任务 `created_by` 可能为空或无法匹配账号，旧机器人没有创建人；无法确定归属的数据必须只对管理员可见，不能猜测后暴露给普通账号。
- OpenSpec CLI 当前环境不可用，本工件按仓库现有 `spec-driven` 结构创建；安装 CLI 后需补做 status、instructions 和格式校验。
