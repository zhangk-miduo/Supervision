# 变更提案：增强调度、账号体系与企业微信集成

## 背景

Supervision 当前是由任务、任务节点、Quartz Cron、执行日志和企微群机器人组成的 MVP。系统没有登录鉴权、后台账号、角色、组织人员、部门或系统设置；定时任务只接受 Cron；企微能力仅为固定 Markdown 的群机器人 Webhook，尚不能同步通讯录、从组织选择人员、按人或部门发送应用消息，也不能真正支持多消息类型。

本次变更将项目升级为可供企业内部正式使用的督办平台基础版本，并把身份、组织、触达和调度拆成边界清晰的四项能力。

## 目标

- 提供可视化的丰富定时规则，并保留高级 Cron 模式。
- 建立独立的后台登录账号体系，支持管理员创建账号、首次登录强制改密、角色权限和登录审计。
- 从企业微信同步部门与人员，禁止后台手工新增组织人员。
- 在系统设置中管理企微自建应用、通讯录同步和群推送配置。
- 在任务中以结构化表单配置企微群消息或应用消息，从组织树选择人员、部门或标签。
- 支持群内 @指定人员和 @所有人，并对群成员不可验证的限制给出明确提示。
- 支持企微文本、Markdown、Markdown v2、图片、图文、文件、语音和模板卡片的可扩展消息模型。
- 记录每次消息投递结果，支持审计、限流、失败原因和后续重试。

## 非目标

- 本变更不实现企微扫码登录，但数据与接口边界须允许后续增加。
- 首期不实现多企业租户，只预留企业标识和配置扩展能力。
- 不实现人员、部门信息反向写回企业微信。
- 不在首期实现复杂法定节假日服务；先交付周一至周五工作日规则和日历扩展点。
- 不提供任意 JSON 作为最终用户的主要消息编辑方式。
- 不自动把企业微信人员创建为后台登录账号。

## 能力范围

1. `advanced-scheduling`：一次、周期、Cron、时区、有效期、错过执行、并发、重试和未来执行时间预览。
2. `account-authentication`：本地账号密码、首次改密、账号与人员可选绑定、角色权限、锁定与审计。
3. `wecom-directory-sync`：企微配置、部门与人员全量同步、停用策略、同步日志和组织选择器数据接口。
4. `wecom-messaging`：群机器人与应用消息双通道、消息类型编辑器、人员选择、@能力、投递记录和安全控制。

## 关键用户价值

- 管理员可用普通业务语言配置可靠的定时任务，不需要理解 Cron。
- 组织人员名单与企业微信保持一致，避免重复维护和脏数据。
- 登录权限与组织人员生命周期解耦，兼顾安全管理员、服务账号和普通员工。
- 任务创建者可以直观选择接收范围和消息样式，并在发送前预览和测试。
- 运维人员可以追踪每次触达是否成功，而不是只看到笼统的任务执行结果。

## 影响范围

- 后端：新增认证、账号、RBAC、组织、企微配置与客户端、同步、消息投递等领域；重构调度和企微节点执行。
- 前端：新增登录、首次改密、账号管理、组织人员、系统设置；重构任务创建为分步表单和结构化消息编辑器。
- 数据库：新增账号、角色、部门、人员、同步日志、企微配置、模板、接收人和投递记录等表；扩展调度表。
- 基础设施：Redis 缓存企微 access_token 和登录状态；Quartz 支持多个 Trigger 与调度策略；敏感配置加密。
- 文档与测试：补充配置指南、权限矩阵、企微联调说明、迁移验证和用户验收清单。

## 验收摘要

- 管理员能创建登录账号，新账号首次登录只能修改密码，修改后才能访问系统。
- 账号可以不绑定人员；同步人员不会自动创建账号。
- 企微配置可验证，部门与人员可全量同步，后台不能新增人员。
- 任务可使用可视化规则配置每天、每周、每月和一次执行，并显示未来五次触发时间。
- 任务可选择群消息或应用消息，选择组织人员或部门，配置消息类型并发送测试。
- 群消息支持 @指定人员和 @所有人；系统明确提示无法确认所选人员是否在目标群中。
- 每次发送都形成可查询的投递记录，Secret、密码和 Webhook 不以明文存储或回显。

## 依赖与风险

- 需要有效的企微企业、自建应用、通讯录权限、AgentId/Secret 和群消息推送地址进行真实联调。
- 企微应用可见范围会影响同步及发送结果，必须在配置验证中呈现。
- 当前前端定时值判断与后端约定不一致，现有企微客户端也固定发送 Markdown，应在迁移第一步修复。
- 群机器人无法可靠获取群成员列表，组织人员存在不等于在目标群内，产品必须保留限制提示。
- OpenSpec CLI 当前环境不可用，本工件按本机既有 OpenSpec 结构生成，后续安装 CLI 后需要执行状态校验。
