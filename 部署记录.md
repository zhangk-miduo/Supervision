## 三、部署过程中修复的问题（已合入代码/编排）

1. **编译错误（Spring Boot 3 Jakarta 迁移）**
   - `javax.annotation.PostConstruct` → `jakarta.annotation.PostConstruct`（SchedulerAppService）
   - `NotificationProducer` / `NotificationConsumer` 缺少 `RabbitMQConfig` 的 import
   - `ExecutionStatus` 的 import 包路径错误（`domain.model` → `domain.model.enumeration`，TaskAppService / TaskExecutionEngine）

2. **RabbitMQ 启动崩溃**
   - 原 `rabbitmq:3.13-alpine` + 废弃环境变量 `RABBITMQ_VM_MEMORY_HIGH_WATERMARK` 会导致容器拒绝启动（exit 1）。
   - 改为 `rabbitmq:3.13-management-alpine`，内存配置移入 `docker/rabbitmq/rabbitmq.conf`，并设置显式账号 `supervision/supervision`（避免 guest 在容器网络被拒）。

3. **RabbitMQ 连接指向 localhost**
   - `spring.rabbitmq.host` 未从 `RABBITMQ_HOST` 占位符取到值，导致 API 连 `localhost:5672` 失败。
   - 在 compose 的 api 环境中增补规范的 `SPRING_RABBITMQ_HOST/PORT/USERNAME/PASSWORD`，确保绑定到 `spring.rabbitmq.*`。

4. **前端容器健康检查 unhealthy**
   - `nginx:alpine` 不含 `wget`，原 HEALTHCHECK 用 `wget` 必失败。改为 `curl -f http://localhost/api/health`。

## 四、运维命令（在服务器 /opt/supervision 下）

```bash
# 启动 / 重建
sudo docker compose up -d --build

# 停止
sudo docker compose down

# 查看状态
sudo docker compose ps

# 查看日志
sudo docker compose logs -f api          # 后端
sudo docker compose logs -f web          # 前端/Nginx

# 仅重建某个服务
sudo docker compose up -d --build api
```
